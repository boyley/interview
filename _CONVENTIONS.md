# interview 面试库 · 统一规范（所有 sub-agent 必须遵守）

> 本文件是 `interview/` 唯一风格标准。任何 agent 生成内容前先读本文件。
> 定位：**考前速刷 + 面试怎么答**。深度原理不在这里展开，链接回 `*-learning` 深度库。

## 一、与 *-learning 深度库的分工

| 库 | 定位 | 内容形态 |
|----|------|----------|
| `jvm-learning` / `java-learning` / … | **深度原理**，系统学习 | 一个知识点一篇长文、Mermaid、源码 |
| `interview/`（本库） | **面试速答**，考前突击 | 短问快答、速记卡、手撕代码、场景题 |

原则：**interview 讲「怎么答、答什么、易错」，深挖某点时用相对链接指向 learning 库**，例如
`详见 [双亲委派](../../jvm-learning/06-parent-delegation.md)`。不要把 learning 库的长文复制过来。

## 二、目录结构

```
interview/
├── README.md            ← 总览 + 复习路线 + 覆盖清单（进度看这里）
├── _CONVENTIONS.md      ← 本文件
├── 01-cheatsheet/       ← 八股速答：每个领域一篇，考前速刷
├── 02-coding/           ← 手撕代码：高频算法 + 手写实现（单例/LRU/线程池…）
├── 03-system-design/    ← 场景/系统设计题
└── 04-project-hr/       ← 项目深挖话术 + 行为/HR 面
```

每个模块目录内有自己的 `README.md`（该模块的 TODO 清单 + 索引），文件用两位数字编号。

## 三、命名

- 文件：`NN-topic.md`（两位数字 + kebab-case 英文），如 `03-jvm.md`、`01-lru-cache.md`。
- 编号 = 模块内推荐刷题/复习顺序。

## 四、各模块 md 固定结构

### 01-cheatsheet（八股速答）—— 追求「一眼扫过、能背出口」

```markdown
# NN · 领域中文名 速答（English）

> 一句话定位 + 本篇覆盖的高频考点数。

## 🔥 高频必背（Top N）
（表格或 Q→一句话答，覆盖最高频 10~20 问，答案压到 1~3 句能背出口。）

## 📌 展开速答
（次高频问题，Q 加粗 + 3~5 句答案；需要深挖处链接 learning 库。）

## ⚠️ 易错 / 反问加分
（常见坑 + 面试官追问时的加分点。）
```

### 02-coding（手撕代码）

```markdown
# NN · 题目名（English）

> 考点 + 难度（🟢简单/🟡中等/🔴困难）+ 出现频率。

## 题目
（题面 + 输入输出示例。）

## 思路
（最优解思路，2~4 句；点明时间/空间复杂度。）

## 代码（Java）
（可编译、带关键注释；手写实现类题目要能默写。）

## 易错点 / 追问
（边界、面试官常见追问：如「能否降到 O(1) 空间」。）
```

### 03-system-design / 04-project-hr

- system-design：需求澄清 → 容量估算 → 核心设计 → 难点权衡 → 演进；配架构图（Mermaid）。
- project-hr：STAR 结构话术模板 + 高频行为问题 + 项目深挖问答。

## 五、内容语言与质量底线

- 讲解一律**中文**；术语/API/命令用英文。
- **面试导向**：多答「怎么答、答什么、易错、加分」，少堆无用细节；能背出口 > 面面俱到。
- 代码可编译、给复杂度；命令给真实可用的。
- 需要深度原理时**链接 learning 库**，不重复长文。
- 每篇聚焦，别写成论文；速答篇尤其要短。
