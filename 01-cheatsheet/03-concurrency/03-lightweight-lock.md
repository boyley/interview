# 轻量级锁详细机制解析

> **核心理念：轻量级锁针对「多线程交替执行同步块、无实际竞争或竞争很轻」的场景，用「Lock Record + CAS + 自旋」避免直接上重量级锁（操作系统 mutex、用户态↔内核态切换）的昂贵开销。**
> 答题方法：先补对象头/锁升级背景 → 加锁机制 → 解锁机制 → 设计动机 → 自旋 → 膨胀时机。

---

## 一、前置：对象头 & Mark Word

对象在内存中有**对象头**，其中 **Mark Word** 存锁信息（也复用存 hashcode/GC 年龄），最后 2~3 位是**锁标志位**：

| 锁状态 | 标志位 | Mark Word 主要内容 |
|---|---|---|
| 无锁 | `01` | hashcode、GC 年龄 |
| 偏向锁 | `01`(偏向位=1) | 偏向的 threadId |
| **轻量级锁** | `00` | **指向栈中 Lock Record 的指针** |
| 重量级锁 | `10` | 指向 monitor(ObjectMonitor) 的指针 |

---

## 二、锁升级全景（定位轻量级锁）

```
无锁 ──► 偏向锁 ──► 轻量级锁 ──► 重量级锁
       (单线程)   (多线程交替、轻竞争)  (激烈竞争)
              ↑ 只能升级不能降级（不可逆）
```

> 轻量级锁 = 偏向锁失效后、竞争还不激烈时的**中间态**。
> （注：偏向锁在 **JDK15 起默认关闭、JDK18 移除**；轻量级锁仍在。）

---

## 三、加锁机制（★ 核心，小步骤）

1. 线程进同步块，锁对象处于**无锁/可轻量化**状态
2. 在**当前线程的栈帧**中开辟一块 **Lock Record（锁记录）**
3. 把锁对象的 **Mark Word 拷贝进 Lock Record**（称 **Displaced Mark Word / 置换标记字**）
4. **CAS** 尝试把对象的 Mark Word 改为**指向该 Lock Record 的指针**（标志位 → `00`）
5. **CAS 成功** → 拿到轻量级锁，执行同步块
6. **CAS 失败** → 有竞争，分两种：
   - **是自己**（重入）：Mark Word 指向的 Lock Record 在自己栈里 → 重入，再压一个**空 Lock Record** 计数
   - **是别人**：有其他线程持有 → **自旋**重试 CAS
   - 自旋超阈值 / 自适应判断没戏 → **锁膨胀升级为重量级锁**，Mark Word 指向 monitor，线程阻塞

```
线程栈帧                        锁对象头
┌──────────────┐               ┌──────────────┐
│ Lock Record  │◄──CAS 指针────│  Mark Word=00 │
│  Displaced   │──拷贝原值──────│  (指向左边)    │
│  Mark Word   │               └──────────────┘
└──────────────┘
```

---

## 四、解锁机制（小步骤）

1. **CAS** 把 Lock Record 里的 **Displaced Mark Word 换回**对象的 Mark Word
2. **CAS 成功** → 释放锁，对象恢复**无锁**状态
3. **CAS 失败** → 说明期间**已膨胀成重量级锁**（有线程自旋失败触发了膨胀）→ 按重量级锁流程释放并**唤醒阻塞的线程**

---

## 五、为什么用轻量级锁（设计动机）

- **重量级锁**靠 OS **mutex**：涉及**用户态↔内核态切换**、线程阻塞/唤醒，非常昂贵
- 经验假设：**"绝大多数锁在整个同步周期内并不存在竞争"**，只是多线程**交替**执行
- 无竞争时用 **CAS 就够**，避免进内核 → 大幅降低开销

---

## 六、自旋 & 自适应自旋

- 竞争时**不立即阻塞**，而是**自旋（空转）**等持有者释放，省掉线程切换开销
- **JDK6+ 自适应自旋**：自旋次数根据**上次在同一锁上自旋的成功率**动态调整（历史成功就多转会儿，老失败就干脆别转直接膨胀）

---

## 七、三种锁对比

| 锁 | 适用 | 机制 | 是否阻塞 |
|---|---|---|---|
| **偏向锁** | 单线程反复进出 | CAS 记一次 threadId | 否 |
| **轻量级锁** | 多线程**交替**/轻竞争 | Lock Record + CAS + 自旋 | 否（自旋） |
| **重量级锁** | **激烈竞争** | monitor + OS mutex | 是（阻塞挂起） |

---

## 八、膨胀时机

- **偏向 → 轻量**：出现第二个线程竞争（偏向锁被撤销）
- **轻量 → 重量**：CAS 自旋失败 / 自旋超阈值 / 已有线程在等待

---

> **核心原则：轻量级锁用「栈帧 Lock Record + Displaced Mark Word + CAS + 自旋」，在无/轻竞争时避免重量级锁的内核态开销；加锁=拷贝 Mark Word 到 Lock Record 再 CAS 指向它，解锁=CAS 换回原值；CAS 失败且自旋无果就膨胀为重量级锁。锁只升不降。**
