# 09 · AI 编程 / AI Coding 面试（AI-Assisted Coding）

> AI 辅助编程已是程序员标配，面试越来越爱问"你怎么用 AI 提效、原理是什么、怎么把关"。本篇从**工具分类、底层原理、高效用法、风险管控、面试话术**五个角度拆解。定位：不是考你会背工具名，而是考你**是否真正把 AI 当生产力工具并能守住质量底线**。

## 🔥 高频必背（Top 15）

| # | 问题 | 一句话答 |
|---|---|---|
| 1 | AI 编程工具分几类？ | ①**补全类**（边打边补，如 Copilot 自动补全）②**对话类**（Chat 里问答/解释/改代码）③**Agentic 类**（自主读改多文件+跑命令+迭代，如 Claude Code、Cursor Agent）。 |
| 2 | 主流工具有哪些？ | GitHub Copilot、Cursor、Claude Code、Windsurf、Cline/Aider（开源）、通义灵码/CodeGeeX（国产）。 |
| 3 | 代码补全底层原理？ | **FIM（Fill-in-the-Middle）**——把光标前后代码当上下文，让模型补中间；加上 **codebase 检索**把相关文件塞进上下文。 |
| 4 | AI 怎么"读懂"整个项目？ | **Codebase Indexing**：把代码分块 → Embedding → 存向量库，提问时**检索相关代码片段**拼进上下文，本质是**对代码做 RAG**。 |
| 5 | 什么是 Agentic Coding？ | AI 自主循环：**读文件→改代码→跑测试/命令→看结果→再改**，直到完成任务；不再是一次补全，而是多步自主执行（Claude Code、Cursor Agent）。 |
| 6 | AI 写代码为什么会出错？ | 会**幻觉**：编造不存在的 API/库、用**过时**的写法、**上下文不足**时瞎猜项目约定、依赖版本对不上。 |
| 7 | 怎么让 AI 生成更准？ | 给**清晰上下文**（相关文件/示例/约束）、**小步迭代**、用**规则文件**（`CLAUDE.md`/`.cursorrules`）固化项目规范、让它**跑测试自验**。 |
| 8 | 规则文件是什么？ | 项目根放 `CLAUDE.md` / `.cursorrules` / `AGENTS.md`，写明技术栈、代码规范、禁忌，AI 每次自动读取，**避免重复交代**。 |
| 9 | MCP 在 AI 编程里干嘛？ | 让 AI 工具**标准化接外部能力**（数据库、Jira、浏览器、文档），不用为每个工具定制集成。详见 [`04-function-calling-mcp`](04-function-calling-mcp.md)。 |
| 10 | AI 生成代码最大风险？ | **质量与安全**：隐藏 bug、安全漏洞、许可证污染（抄了 GPL 代码）、**把私有代码/密钥泄露**给云端模型。 |
| 11 | 用 AI 就不用 review 了吗？ | **恰恰更要 review**。AI 是"超快的初级工程师"，产出必须人工把关；不 review 就合并是最大的坑。 |
| 12 | 什么是 Vibe Coding？ | 用自然语言"感觉着"让 AI 生成大段代码、少看细节的编程方式；快但**质量不可控**，只适合原型，不适合生产核心。 |
| 13 | AI 适合写什么、不适合写什么？ | 适合：样板代码、单测、正则、脚本、格式转换、查文档、解释报错；不适合：核心业务逻辑、涉密/高安全、需深度架构权衡的部分——这些 AI 起草、人定夺。 |
| 14 | 企业怎么防代码外泄？ | **私有化/本地部署模型**、企业版（通义灵码企业版/Copilot Enterprise）、脱敏、禁止把密钥和核心代码喂云端。 |
| 15 | AI 编程对开发者的影响？ | 从"**写代码**"转向"**描述需求 + review + 架构设计**"；提效明显，但**基本功和判断力更重要**（要能看出 AI 哪里错）。 |

## 🧰 主流工具对照

| 工具 | 类型 | 特点 | 定位 |
|---|---|---|---|
| **GitHub Copilot** | 补全 + Chat + Agent | 起步最早、IDE 集成最广、生态成熟 | 通用主力 |
| **Cursor** | AI-first 编辑器（Agent） | 深度 codebase 理解、Composer 多文件改、`.cursorrules` | 重度 agentic 编辑 |
| **Claude Code** | CLI Agentic | 终端里自主读改跑、强 agentic、`CLAUDE.md`、MCP、Hooks | 终端/自动化流 |
| **Windsurf** | AI-first 编辑器 | Cascade 流式 agent、上下文感知 | 类 Cursor |
| **Cline / Aider** | 开源 Agentic | 可接任意模型、透明可控 | 自托管/省钱 |
| **通义灵码 / CodeGeeX** | 补全 + Chat | 国产、国内直连、可私有化部署 | 国内企业合规 |

> 趋势：**从"补全"走向"Agentic"**——过去是"帮你补下一行"，现在是"给个任务，自己读改跑测试交活儿"。

## 🔄 Agentic Coding 的工作循环

```
任务（自然语言）
   │
   ▼
[理解] 读相关文件 / 检索 codebase（对代码做 RAG）
   │
   ▼
[规划] 拆成步骤
   │
   ▼
┌─►[执行] 改代码 / 建文件
│     │
│     ▼
│  [验证] 跑测试 / 编译 / 执行命令
│     │
│     ▼
└──[观察] 有错？→ 读报错 → 再改（循环，带最大步数上限）
      │ 通过
      ▼
[交付] 汇报改动，等人 review
```

> 和普通补全的本质区别：**补全是"一次性猜下一段"，Agentic 是"多步自主执行 + 用工具自验"**——这正是 [`05-agent`](05-agent.md) 讲的 Agent 那套（ReAct 循环）落到编程场景。

## 📌 展开速答

**Q：AI 是怎么"读懂"我整个项目的？（原理题）**
两个机制：①**Codebase Indexing（代码 RAG）**——把项目代码切块、Embedding、存进向量库；你提问时，工具先**语义检索**出最相关的代码片段，连同你的问题一起塞进模型上下文。这就是为什么它能"知道"你项目里某个类/函数，本质是 [`03-rag`](03-rag.md) 那套 RAG 用在代码上。②**主动读文件（Agent）**——agentic 工具会像人一样用工具**grep/读文件/看目录**按需抓上下文，而不是靠预先索引。现代工具两者结合。局限：上下文窗口有限，超大项目仍可能"看漏"，所以给它**指明关键文件**能显著提准。

**Q：代码补全（Copilot 那种）底层是什么原理？**
核心是 **FIM（Fill-in-the-Middle，中间填充）**：模型训练时学过"给定光标前缀 + 后缀，预测中间该填什么"，所以能结合上下文补全而非只顺着往下写。再叠加**上下文检索**（把当前文件、打开的相邻文件、检索到的相关代码拼进 prompt）提升相关性。它仍是**逐 token 生成**，所以会有幻觉——补出看似合理但项目里根本不存在的方法名，这也是必须人 review 的原因。

**Q：AI 生成代码会犯哪些典型错误？（会答=用过且踩过坑）**
①**幻觉 API**：编造不存在的方法/参数/库（最常见）；②**过时写法**：训练数据有截止，用了已废弃的 API 或旧版本语法；③**依赖/版本错配**：import 了项目没装的包，或版本对不上；④**忽略项目约定**：上下文不够时按"通用写法"猜，不符合你的分层/命名/框架规范；⑤**看似对实则错**：能编译能跑，但边界条件/并发/安全有隐藏 bug；⑥**过度自信**：把不确定的东西写得很笃定。**治理靠：给足上下文 + 规则文件固化规范 + 让它跑测试自验 + 人工 review 每个 diff。**

**Q：怎么高效使用 AI 编程工具？（最能体现你会用）**
一套实践：①**给清晰上下文**——指明相关文件、给个类似实现当范例、说清约束（用什么库/不准改什么）；②**小步迭代**——一次一个明确任务，别让它一口气改一堆，diff 小才好 review；③**规则文件**——`CLAUDE.md`/`.cursorrules` 写死技术栈和规范，省得每次交代、也让产出更一致；④**让它自验**——要求跑测试/编译，agentic 工具能自己看报错迭代；⑤**始终 review**——把 AI 当"手很快的初级同事"，产出必看、必测、必要时打回重写；⑥**善用 MCP**——接数据库/文档/浏览器等外部上下文。核心心法：**你负责"想清楚要什么 + 把关质量"，AI 负责"快速产出草稿"。**

**Q：AI 生成代码的安全与合规风险怎么管？**
①**代码泄露**：把私有代码/密钥/客户数据喂给云端模型有外泄风险 → 用**私有化部署/企业版**、脱敏、`.gitignore`/密钥隔离；②**安全漏洞**：AI 可能生成有注入/越权/硬编码密钥的代码 → 过 SAST/安全扫描 + review；③**许可证污染**：可能复现训练数据里的 GPL 等受限代码 → 用带**过滤/溯源**的企业版、关键代码人工确认；④**质量债**：无脑合并 AI 代码会累积隐性 bug → 强制 review + 测试门禁。**一句话：AI 提效不改变"谁提交谁负责"，把关责任在人。**

**Q：面试官问"你平时怎么用 AI 提效"，怎么答出彩？**
别泛泛说"我用 Copilot 写得快"。用**具体场景 + 把关**：①"重复样板/CRUD/DTO 转换/正则这类**低价值高重复**的活交给 AI，我省下时间做设计"；②"写**单元测试**、补边界用例效率翻倍，但断言我会审"；③"**读陌生代码库/查报错/查 API 用法**当交互式文档，比翻文档快"；④"**重构**时让它先出方案我再定夺"；⑤强调**我 review 每个改动、跑测试验证、核心逻辑自己把关，不盲信 AI**。最后点一句"**AI 让我从写代码更多转向想清楚需求和架构**"——既显得会用，又显得有判断力，这是加分核心。

## ⚠️ 易错 / 反问加分

- ⚠️ **别显得"盲信 AI"**——面试官最怕你无脑复制 AI 代码。任何时候都要带一句"我会 review / 测试 / 把关"，否则减分。
- ⚠️ **别把 AI 编程说成"取代程序员"**——正确姿势是"放大程序员产能，但基本功和判断力更关键（要能看出它哪错）"。
- ⚠️ **Vibe Coding 别当褒义词乱用**——它适合快速原型/玩具，用在生产核心逻辑上是反面教材，答题要点明适用边界。
- ✅ **加分**：讲原理时点出"**codebase indexing 就是对代码做 RAG**、agentic coding 就是 **ReAct 循环 + 工具自验**"——把 AI 编程接回 [`03-rag`](03-rag.md)/[`05-agent`](05-agent.md) 的原理，显示你懂底层不只是会用。
- ✅ **加分**：提**规则文件（CLAUDE.md/.cursorrules）和 MCP**——说明你会"配置项目上下文让 AI 更准"，是深度用户而非浅尝。
- ✅ **加分**：谈企业落地时点**私有化部署/代码不外泄/许可证合规**，体现你有工程/安全/合规意识，不只关心"好不好用"。
- ✅ **加分**：一句"**AI 把编码成本压低后，瓶颈从'写得快'转移到'需求想得清、代码 review 得住、系统设计得对'**"——格局拉满。
