## 🎯 面试冲刺清单（必须能张口就答）

- **Token 是什么、为什么按 Token 计费、上下文窗口满了怎么办**（记忆截断/摘要压缩/RAG）。
- **幻觉是什么、为什么会有、工程上怎么压**（RAG 提供依据 + 降温度 + 结构化约束 + 引用溯源 + 评估兜底）。
- **RAG 全流程能画出来**：文档 → 分块 → Embedding → 存向量库 → 检索 Top-K → 拼进 Prompt → 生成。以及每一步的坑（分块粒度、检索召回、重排）。
- **Function Calling 的本质**：模型不执行代码，只**输出"该调哪个函数+参数"的 JSON**，由你的程序执行后把结果再喂回去（多轮）。
- **MCP 是什么**：给"模型接工具/数据源"定的**标准协议**，让工具可插拔、跨框架复用（类比 USB-C / JDBC）。
- **Agent 为什么不稳、怎么兜底**：多步自主决策 → 错误累积 → 要加最大步数、超时、人工确认、可观测。
- **AI 应用的工程难点**：慢（流式+异步）、贵（缓存+限流+小模型分流）、不确定（结构化输出+校验+重试）、会瞎编（RAG+评估）、不安全（Prompt 注入+内容审核）。
- **框架选型**：Spring AI（Spring 生态/自动配置/ChatClient）vs LangChain4j（对标 Python LangChain/原生 API/AI Services），能说出各自定位。

## ⚠️ 程序员答 AI 面试的通用加分/避坑

- ✅ **加分**：任何 AI 功能都从「不确定/慢/贵/会瞎编/不安全」五个工程角度谈治理，立刻显得像做过生产而非只跑过 Demo。
- ✅ **加分**：能把 AI 概念映射回熟悉的后端概念——Function Calling≈RPC、MCP≈JDBC/USB-C、向量检索≈带索引的相似度查询、Advisor≈拦截器/AOP、RAG≈"开卷考试"。
- ⚠️ **坑**：别把自己包装成算法岗，被追问 Transformer 内部/训练细节会露馅；应用岗答"我负责调用与工程化落地"是加分不是减分。
- ⚠️ **坑**：说 RAG 别只会背"检索+生成"，要能讲**每步的坑与优化**（分块、召回率、重排、上下文超长），否则一问就穿。
